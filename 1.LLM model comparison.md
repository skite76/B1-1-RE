# LLM 모델 비교·선정 보고서  
**주제: 행복마을 이웃사촌 축제 회의록 요약 과업 비교**

## 1. 평가 목적

동일한 회의 원문과 단계별 프롬프트를 바탕으로, 세 가지 LLM 모델의 응답 품질을 비교했습니다.

비교 대상 모델은 다음과 같습니다.

1. **Claude Opus 4.8**
2. **Gemini 3.1 Pro**
3. **GPT-5.5**

평가의 핵심은 주민들에게 제공할 최종 회의록 요약본을 만들 때 어떤 모델이 가장 적합한지 판단하는 것입니다.

---

## 2. 주요 평가 기준

| 평가 기준 | 설명 |
|---|---|
| 정확성 | 회의 원문에 있는 사실을 왜곡하지 않는가 |
| 수치 검증 능력 | 원문에 있는 수치와 없는 수치를 구분하는가 |
| 간결성 | 필요한 내용만 짧고 명확하게 정리하는가 |
| 지시 준수 | 사용자가 요청한 출력 형식과 조건을 잘 따르는가 |
| 친화적 톤 | 주민들이 불편하지 않도록 부드럽고 정중하게 표현하는가 |
| 조건부 판단 | “전력 충분 + 렌탈비 10% 이내” 조건을 적절히 반영하는가 |
| 최종 활용성 | 실제 주민 안내문으로 바로 활용 가능한가 |

---

## 3. 테스트 질문별 모델 결과 비교

| 테스트 과업 | Claude Opus 4.8 | Gemini 3.1 | GPT-5.5 |
|---|---|---|---|
| 1. 기본 회의록 요약 | 전반적으로 정확하고 깔끔함 | 비교적 자세하고 친절함 | 핵심 구조가 명확하고 정확함 |
| 2. 구체적 수치 활용 요약 | 원문 수치 중심으로 비교적 안전하게 작성 | “상생 100%”, “수거율 100%” 등 원문에 없는 수치 사용 | 원문 수치와 일정 중심으로 비교적 안정적 |
| 3. 수치 정확성 검증 | 원문 수치와 없는 수치를 잘 구분함 | 원문에 없는 수치를 스스로 인정하고 수정함 | 원문 수치, 계산 수치, 없는 수치를 구분함 |
| 4. 원문에 없는 수치 제시 | 총예산, 목표 인원, 부스 개수 등 잘 지적 | 임의로 넣은 0%, 100% 표현을 명확히 지적 | 운영 수치 전반을 폭넓게 정리함 |
| 5. 원문 수치만 사용한 재요약 | 정확성은 좋으나 일부 항목은 다소 보수적으로 처리 | 수정 후 정확도 개선 | 가장 안정적으로 정리함 |
| 6. 예산 1억 원일 때 렌탈비 계산 | 4,000만 원으로 정확히 계산 | 4,000만 원으로 정확히 계산 | 4,000만 원으로 정확히 계산 |
| 7. 결정사항과 Action Items 추출 | 박현우 업무만 포함해 일부 누락 | 역할·직책 오류와 추정 업무가 섞임 | 확실한 업무 위주로 정리해 비교적 적절함 |
| 8. 담당자별 제안/업무 추출 | 해당 테스트 기록 없음 | 제안자와 담당자 일부를 잘못 배정 | 박현우 제안 내용을 누락하는 등 오류 있음 |
| 9. 행사내용/문제점/대안 요약 | 원문에 있는 문제도 확인 필요로 돌리는 경향 | 구조는 좋으나 마술 공연을 확정 대안처럼 표현 | 비교적 간결하고 요청 형식에 잘 맞음 |
| 10. 친절하고 자상한 톤 변환 | 주민 친화적이고 부드러움 | 매우 부드럽지만 다소 장황함 | 정중하고 간결한 편 |
| 11. 에어바운스 조건부 재검토 | 조건 충족 시 재검토 가능하다고 적절히 반영 | 조건 충족 시 설치 가능성을 강하게 표현 | “다시 검토”로 안전하게 표현 |
| 12. 최종 주민용 요약본 | 부드럽고 주민 친화적이나 일부 확인 필요 표현 부정확 | 친절하지만 원문 확정 사항과 가정이 일부 섞임 | 가장 균형 잡힌 최종 요약본 작성 |

## 4. 모델별 장단점 분석

### 4.1 Claude Opus 4.8

#### 장점
- 전체적으로 문장이 부드럽고 주민 친화적입니다.
- 원문에 없는 수치를 함부로 넣지 않으려는 태도가 좋습니다.
- 조건부 검토 요청을 비교적 안전하게 반영했습니다.
- 최종 요약본의 톤이 정중하고 안정적입니다.

#### 아쉬운 점
- 확실히 원문에 있는 내용도 “확인 필요”로 돌리는 경우가 있었습니다.
  - 예: 먹거리 문제점인 상권 침해 우려
  - 예: 쓰레기 문제인 배달 용기 증가 우려
- Action Items에서 이지아의 포스터 시안 제작 등 명확한 업무를 일부 누락했습니다.
- 메시지 카드 제작은 원문상 이지아가 담당한다고 되어 있는데, “담당자 확인 필요”로 처리한 점은 부정확합니다.

#### 종합 평가
안전하고 친절하지만, 일부 정보를 지나치게 보수적으로 판단하여 누락하는 경향이 있습니다.

---

### 4.2 Gemini 3.1

#### 장점
- 주민 안내문처럼 읽기 쉬운 표현을 잘 만듭니다.
- 친절하고 따뜻한 톤이 강합니다.
- 사용자의 지적을 받은 뒤 원문에 없는 수치를 인정하고 수정하는 능력이 있습니다.
- 설명이 풍부해 학습용 피드백에는 적합합니다.

#### 아쉬운 점
- 초반에 원문에 없는 수치를 마케팅 표현처럼 사용했습니다.
  - “상권 침해 0%”
  - “상생 100%”
  - “수거율 100%”
- 일부 직책과 담당자를 잘못 표기했습니다.
  - 예: 김민수를 관리소장으로 표현
  - 예: 박현우를 동대표로 표현
- 담당자별 제안 내용에서 발화자 배정 오류가 있었습니다.
- 최종 요약본에서 에어바운스 조건부 설치를 다소 확정적으로 표현했습니다.

#### 종합 평가
문장은 가장 친절하지만, 사실 정확성과 원문 충실도 측면에서 위험 요소가 가장 큽니다.

---

### 4.3 GPT-5.5

#### 장점
- 전체적으로 정확성과 간결성의 균형이 가장 좋습니다.
- 원문에 있는 수치와 없는 수치를 잘 구분했습니다.
- 전체 예산 1억 원 기준 렌탈비 계산도 정확했습니다.
- 최종 주민용 회의록 요약본이 가장 실무적으로 활용하기 좋았습니다.
- 조건부 검토를 “설치 확정”이 아니라 “다시 검토”로 표현해 안전했습니다.
- 최종본에 회의 일시, 다음 회의, 행사 내용, 고려 사항, 대안, 담당 업무를 균형 있게 포함했습니다.

#### 아쉬운 점
- 담당자별 제안 내용 추출에서는 오류가 있었습니다.
  - 박현우가 제안한 등불 축제, 배달 존, 지하 주차장 활용 등을 충분히 반영하지 못했습니다.
- 메시지 카드 제작은 원문상 이지아가 담당한다고 볼 수 있는데, 최종본에서는 담당자 확인 필요로 처리했습니다.
- 김민수의 상가 번영회 협의 업무는 원문에 있으나 최종본에서 빠졌습니다.

#### 종합 평가
일부 세부 오류는 있으나, 최종 주민 안내문 작성 과업에서는 세 모델 중 가장 안정적이고 실무 활용성이 높습니다.
